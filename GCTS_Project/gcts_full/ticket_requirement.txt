4. Adjust GCTS Harmonization pipeline - FACT_GCTS_RESPONSE
https://jira.app.pconnect.biz/browse/DODECOM-7402
Adjust matillion transformation job: trn_v_int_fact_gcts_response
Adjust how to get data from T_GCTS_RESPONSE:
Check MAX(_ETL_LOAD_DATETIME) for group by YEAR,MONTH,COUNTRYCATEGORYID
If YEAR,MONTH,COUNTRYCATEGORYID has MAX(_ETL_LOAD_DATETIME) < '2026-04-01' then get the data as is from T_GCTS_RESPONSE.
If YEAR,MONTH,COUNTRYCATEGORYID has MAX(_ETL_LOAD_DATETIME) >= '2026-04-01' then only get the data from T_GCTS_RESPONSE based on MAX(_ETL_LOAD_DATETIME)
union point 2 and 3.
Point 2 explanation:
In old data that we get from Kantar - we might have multiple ETL_LOAD_DATETIME for 1 combination of YEAR,MONTH,COUNTRYCATEGORYID, and we need to keep it like that.
For the new data from Kantar - based on the ingestion process - each YEAR,MONTH,COUNTRYCATEGORYID will be replaced with the new data that came from Kantar, hence we only need the latest data.
